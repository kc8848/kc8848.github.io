Linux 系统更新与升级指南

⚠️ 提示： 在执行系统升级前，请务必备份重要数据，防止因内核更新导致的不兼容问题。

Linux 系统的升级通常分为两步：更新软件包列表（告诉系统有哪些新版本）和执行实际升级（下载并安装新版本）。

一、 Ubuntu / Debian 系 (使用 apt)

适用于 Ubuntu, Debian, Linux Mint, Kali Linux 等。

# 1. 更新软件包列表
sudo apt update

# 2. 升级已安装的软件包
sudo apt upgrade -y

# 3. 升级系统版本及处理依赖（可选）
sudo apt full-upgrade

二、 CentOS / RHEL / Fedora (使用 yum/dnf)

适用于 CentOS 7/8, RHEL, Fedora, AlmaLinux, Rocky Linux。

# CentOS 7 使用 yum
sudo yum update -y

# CentOS 8+ / Fedora 使用 dnf
sudo dnf update -y

三、 核心命令区别解读

Update (更新)

这并不会安装新软件，它只是同步 /etc/apt/sources.list 中的源，获取最新的软件版本信息。

Upgrade (升级)

这会对比本地已安装软件和刚才获取的列表，下载并安装所有有新版本的软件包。

四、 升级后的清理 (推荐)

升级完成后，系统中可能会留下一些过时的安装包或不再需要的依赖项，可以手动清理：

Ubuntu/Debian 清理命令：

sudo apt autoremove && sudo apt autoclean

CentOS/RHEL 清理命令：

sudo yum clean all

五、 如何检查是否需要重启？

如果升级了 Linux 内核 (Kernel)，通常需要重启系统才能生效：

ls /var/run/reboot-required

如果该文件存在，说明你需要执行 sudo reboot。
