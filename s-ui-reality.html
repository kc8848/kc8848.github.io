VLESS + Reality

Reality (Bless) 核心配置指南

Reality 协议不需要你自己购买域名或申请证书。它通过“真实现身”技术，借用他人的合规域名（如 Microsoft 或 Samsung）来通过防火墙检测。

1. 获取密钥 (Keys)

在面板配置 Reality 之前，你需要生成一对公钥和私钥。在服务器 SSH 终端输入：

s-ui x25519

你会得到 Private Key (私钥) 和 Public Key (公钥)。请妥善保存。

2. 面板参数设置

在 s-ui 面板点击“添加入站列表”，选择 VLESS 协议，并开启 Reality 开关：

端口推荐使用 443 (模拟真实 HTTPS)
传输配置选择 TCP
安全传输 (TLS)选择 Reality
目标网站 (Dest)://microsoft.com (或本地访问顺畅的国外大厂)
服务器名称 (SNI)://microsoft.com
Short ID随机填写 8-16 位 16 进制字符 (如: 0123456789abcdef)
Private Key填入刚才生成的 私钥

3. 关键点：什么是 Bless？

在配置项中，Bless（在某些版本面板中称为“流控”或“Flow”）通常建议选择：

xtls-rprx-vision
这是目前最推荐的选择，它能有效消除 TLS 在 TLS 环境中的特征，显著提升隐蔽性。

4. 客户端配置注意

  • 公钥 (Public Key)： 客户端必须填入刚才生成的公钥（不是服务器填的那个私钥）。
  • Fingerprint： 选择 chrome 或 firefox。
  • SpiderX： 保持默认或留空即可。
